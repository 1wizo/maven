Partie 1 · Installer Office LTSC

L'approche peut paraître détournée, mais c'est la plus propre. Au lieu d'un ISO cracké, on utilise l'Office Deployment Tool (ODT) de Microsoft — le même outil que les départements IT. La différence : on le pointe vers le canal Volume License, ce qui nous donne la build LTSC que MAS peut activer.

1

Récupérer l'Office Deployment Tool

Allez sur la page officielle de Microsoft :
URL
https://www.microsoft.com/download/details.aspx?id=49117
Cliquez sur Download, lancez officedeploymenttool_*.exe, acceptez la licence, et extrayez dans un dossier comme C:\OfficeInstall\. Vous obtiendrez setup.exe (à garder) et des XML d'exemple à ignorer.
2

Créer votre configuration

C'est ici que vous dites à l'installateur ce que vous voulez. Microsoft a un outil web pour ça :
URL
https://config.office.com/deploymentsettings
Connectez-vous avec un compte Microsoft. Voici ce qu'il faut régler :

Cliquez sur Export → XML → enregistrez sous configuration.xml à côté de setup.exe.

3

Lancer l'installation

Ouvrez l'Invite de commandes en admin et lancez :
CMD (admin)
cd /d C:\OfficeInstall
setup.exe /configure configuration.xml
Une petite fenêtre Office s'ouvre et commence le téléchargement. Allez faire un café — ça tire 2–4 Go depuis le CDN de Microsoft. Comptez 5–20 minutes. Quand c'est fini, les applis Office apparaissent dans le menu Démarrer.
4

Vérification rapide

Ouvrez Word. Il devrait se lancer sans demander de clé. Vous verrez une bannière « Activer Office » — c'est normal, on s'en occupe dans la Partie 3.
Office LTSC installé
Installation terminée. Continuez avec l'Activation de Windows ou d'Office.

Pour 2024, remplacez Channel="PerpetualVL2021" par "PerpetualVL2024".

Partie 2 · Activer Windows

On utilise MAS (Microsoft Activation Scripts) — un projet open-source qui génère une licence numérique légitime liée au matériel. La méthode HWID crée une licence permanente liée à votre matériel, survivant aux réinstallations, mises à jour et changements de hardware.

1

Ouvrir PowerShell en admin

Appuyez sur Win, tapez powershell, clic droit → Exécuter en tant qu'administrateur. Cliquez sur Oui à l'UAC.
2

Lancer MAS

Collez ça et Entrée :
PowerShell (admin)
irm https://get.activated.win | iex
Bloqué ?
Si l'URL est bloquée, utilisez le miroir : irm https://massgrave.dev/get | iex
3

Choisir la méthode

Tapez 1 → Entrée (Activer Windows), puis choisissez :
1
HWID Recommandé
Licence numérique permanente. Idéal pour Windows 10/11.
2
KMS38 Jusqu'en 2038
Si HWID échoue ou pour Windows Server.
3
Online KMS 180 jours
Pour Windows 7/8.1 uniquement.

Choisissez HWID, confirmez avec Y, attendez « Activation successful ».

4

Vérifier

Tapez 5 → Entrée. Doit dire Windows is permanently activated.
Windows activé
La licence est liée à votre matériel et survit aux mises à jour.

Partie 3 · Activer Office

Office est installé mais pas activé. On utilise Ohook — il patche la vérification d'activation d'Office au niveau binaire. Permanent, hors-ligne, survit aux mises à jour. La méthode la plus fiable de MAS.

1

Fermer toutes les applis Office

Assurez-vous que Word, Excel, PowerPoint, Outlook, Visio, Project sont tous fermés. Vérifiez la zone de notification et quittez les icônes Office. Ohook modifie des fichiers qu'Office verrouille quand il tourne.
2

Relancer MAS

Même lanceur :
PowerShell (admin)
irm https://get.activated.win | iex
3

Activer

  • 2 → Entrée (Activer Office)
  • 1 → Entrée (Ohook)
  • Y → confirmer

Attendez Activation successful.

4

Vérifier

Tapez 5 → Entrée. Tous les produits Office doivent afficher LICENSED. Ouvrez Word → Fichier → Compte — pas de bannière.
Office activé
C'est fini. Ohook est permanent et marche hors-ligne.

Partie 4 · Alternatives Office sur Linux

Si vous êtes sur Linux, Office ne tourne pas nativement. Vous avez deux options solides. OnlyOffice est une suite Linux native, gratuite, qui gère le .docx mieux que tout. LinOffice fait tourner le vrai MS Office 2024 dans une VM Windows — plus lourd, nécessite 8 Go RAM + KVM, mais compatibilité 100%.

Laquelle choisir ?
Honnêtement, pour 90% des utilisateurs Linux, OnlyOffice est la réponse. Léger, gratuit, pas de VM. Passez à LinOffice seulement si vous avez absolument besoin du vrai MS Office.

Comparaison détaillée

# After deps, download repo, run:
chmod +x setup.sh && ./setup.sh
# OR
python3 gui/linoffice.py

Idéal pour : Quand vous avez absolument besoin du vrai MS Office.

Configuration requise LinOffice
x86_64 uniquement + KVM dans le BIOS + 8 Go RAM + 64 Go disque + ~8 Go téléchargement. ~15 min d'install. ⚠ Évitez FreeRDP v3.23 — bug connu.
▸ Commandes CLI LinOffice (après installation)
linoffice CLI
linoffice word              # launch Word
linoffice excel             # launch Excel
linoffice powerpoint        # launch PowerPoint
linoffice onenote           # launch OneNote
linoffice outlook           # launch Outlook
linoffice windows           # full Windows desktop
linoffice update            # update Windows + Office
linoffice stopcontainer     # stop VM (keeps data)

L'installateur ajoute linoffice à votre PATH et crée les .desktop — les applis Office apparaissent dans votre lanceur.

Et Wine/Bottles/CrossOver ?
Vous pouvez faire tourner Office via Wine, mais c'est fragile — les mises à jour cassent tout. CrossOver est payant. LinOffice est la solution gratuite la plus propre.

Dépannage

De vrais problèmes que j'ai réellement rencontrés, et comment les corriger :

CatégorieSymptômeSolution
OfficeL'installateur planteAssurez-vous que configuration.xml est dans le même dossier que setup.exe. Réexportez depuis config.office.com.
OfficeBloque à « Downloading »Problème réseau. Coupez VPN/proxy, autorisez *.officecdn.microsoft.com, vérifiez 5+ Go libres.
OfficeOffice 365 au lieu de LTSCMauvais canal. Mettez « Office LTSC 2021 Perpetual Enterprise ».
OfficeVisio/Project pas installéAjoutez-les comme entrées séparées dans le XML.
MASNe résout pas get.activated.winDNS qui bloque. Utilisez le miroir : irm https://massgrave.dev/get | iex
MASDefender supprime le scriptFaux positif attendu. Désactivez la protection temps réel, lancez MAS, réactivez.
MASHWID échoue (0xC004C003)Essayez KMS38. Ou MAS → [6] Troubleshoot → Fix Licensing.
MASOffice non détectéBuild grand public (C2R), pas Volume License. Reconstruisez avec PerpetualVL2021.
MASDit « 180 jours » pas permanentVous avez Online KMS. Relancez, prenez HWID pour Windows, Ohook pour Office.
MASActivation perdue après changement matérielHWID lié à la carte mère + CPU. Relancez HWID — reconnexion auto.
Réseau0x80072EE2 / 0x80072EFDConnectivité. Coupez VPN, autorisez *.windows.com et *.microsoft.com, essayez DNS 1.1.1.1.
Vérification manuelle
Lancez slmgr /xpr dans CMD pour Windows, cscript ospp.vbs /dstatus depuis le dossier Office pour Office.
