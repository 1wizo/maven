Part 1 · Installing Office LTSC

The approach might feel roundabout, but it's the cleanest way. Instead of a cracked ISO, we use Microsoft's own Office Deployment Tool (ODT) — the same tool enterprise IT uses. The difference: we point it at the Volume License channel, which gives us the LTSC build that MAS can activate.

1

Grab the Office Deployment Tool

Head to Microsoft's official download page:
URL
https://www.microsoft.com/download/details.aspx?id=49117
Click Download, run officedeploymenttool_*.exe, accept the license, and extract to a folder like C:\OfficeInstall\. You'll get setup.exe (keep this) and some sample XML files you can ignore.
2

Build Your Configuration

This is where you tell the installer what you want. Microsoft has a web-based tool for this:
URL
https://config.office.com/deploymentsettings
Sign in with any Microsoft account. Here's what to set:
Office Products
Products
Office LTSC Professional Plus 2021 - Volume License
Update channel
Office LTSC 2021 Perpetual Enterprise — this is critical
Apps
Tick Word, Excel, PowerPoint, Outlook, OneNote. Untick Clipchamp, Bing, junk.
Language
Language
English (US) / French / Arabic / Spanish / German
Settings
Installation platform
64-bit
Update automatically
Yes
Pin to taskbar
No
Remove older versions
Yes
Licensing
Accept EULA
Yes
Product key
leave blank
AUTOACTIVATE
FALSE — MAS handles this

Click Export → XML → save as configuration.xml next to setup.exe.

3

Run the Installer

Open Command Prompt as administrator and run:
CMD (admin)
cd /d C:\OfficeInstall
setup.exe /configure configuration.xml
A small Office window pops up and starts downloading. Go make coffee — it's pulling 2–4 GB from Microsoft's CDN. Takes 5–20 minutes. When done, Office apps appear in your Start menu.
4

Quick Verification

Open Word. It should launch without asking for a product key. You'll see an "Activate Office" banner — that's expected, we'll fix it in Part 3.
Office LTSC installed
Installation complete. Continue to Windows Activation or Office Activation.
▸ Want to write the XML by hand? Here are the product IDs
configuration.xml
<Configuration>
  <Add OfficeClientEdition="64-bit" Channel="PerpetualVL2021">
    <Product ID="ProPlus2021Volume">
      <Language ID="en-us" />
    </Product>
    <Product ID="Visio2021Volume">
      <Language ID="en-us" />
    </Product>
    <Product ID="Project2021Volume">
      <Language ID="en-us" />
    </Product>
  </Add>
  <Display Level="Full" AcceptEULA="TRUE" />
  <Property Name="AUTOACTIVATE" Value="FALSE" />
</Configuration>
ProPlus2021Volume— Office LTSC Pro Plus 2021
Standard2021Volume— Office LTSC Standard 2021
Visio2021Volume— Visio LTSC 2021
Project2021Volume— Project LTSC 2021
ProPlus2024Volume— Office LTSC Pro Plus 2024
Visio2024Volume— Visio LTSC 2024
Project2024Volume— Project LTSC 2024

For 2024 versions, swap Channel="PerpetualVL2021" to "PerpetualVL2024".

Part 2 · Activating Windows

We're using MAS (Microsoft Activation Scripts) — an open-source project that generates a legitimate hardware-bound digital license. The method is HWID: it creates a permanent license tied to your hardware, surviving reinstalls, updates, and most hardware changes.

1

Open PowerShell as Administrator

Press Win, type powershell, right-click → Run as administrator. Click Yes on UAC.
2

Run the MAS Launcher

Paste this and press Enter:
PowerShell (admin)
irm https://get.activated.win | iex
Blocked?
If the URL is blocked, use the mirror: irm https://massgrave.dev/get | iex
3

Choose Your Method

Type 1 → Enter (Activate Windows), then choose:
1
HWID Recommended
Permanent digital license. Best for Windows 10/11.
2
KMS38 Until 2038
Use if HWID fails or for Windows Server.
3
Online KMS 180 days
For Windows 7/8.1 only.

Pick HWID, confirm with Y, wait for "Activation successful".

4

Verify

Type 5 → Enter. Should say Windows is permanently activated.
Windows activated
The license is tied to your hardware and survives updates.

Part 3 · Activating Office

Office is installed but not activated. We'll use Ohook — it patches Office's activation check at the binary level. Permanent, offline, survives updates. The most reliable method MAS offers.

1

Close All Office Apps

Make sure Word, Excel, PowerPoint, Outlook, Visio, Project are all closed. Check the system tray and quit any Office icons. Ohook modifies files that Office locks while running.
2

Run MAS Again

Same launcher:
PowerShell (admin)
irm https://get.activated.win | iex
3

Activate

  • 2 → Enter (Activate Office)
  • 1 → Enter (Ohook)
  • Y → confirm

Wait for Activation successful.

4

Verify

Type 5 → Enter. All Office products should show LICENSED. Open Word → File → Account — no activation banner.
Office activated
Done. Ohook is permanent and works offline.

Part 4 · Linux Office Alternatives

If you're on Linux, Office doesn't run natively. You've got two solid options. OnlyOffice is a native Linux suite that's free and handles .docx better than anything. LinOffice runs the real MS Office 2024 in a Windows VM — heavier, needs 8 GB RAM + KVM, but 100% compatibility.

Which one should you use?
Honestly, for 90% of Linux users, OnlyOffice is the answer. Light, free, no VM. Only go with LinOffice if you absolutely need real MS Office (VBA macros, perfect Excel rendering, Outlook for Exchange).

What about Wine/Bottles/CrossOver?
You can run Office through Wine, but it's fragile — updates break it. CrossOver is paid. LinOffice is the cleanest free solution if you really need real Office.

Troubleshooting

Real problems I've actually seen, and how to fix them:

AreaSymptomFix
OfficeInstaller errors outMake sure configuration.xml is in the same folder as setup.exe. Re-export from config.office.com if malformed.
OfficeHangs at "Downloading"Network issue. Disable VPN/proxy, allow *.officecdn.microsoft.com, check 5+ GB free.
OfficeGot Office 365 instead of LTSCWrong update channel. Set it to "Office LTSC 2021 Perpetual Enterprise".
OfficeVisio/Project didn't installAdd them as separate entries in the XML.
MASCan't resolve get.activated.winDNS blocking. Use mirror: irm https://massgrave.dev/get | iex
MASDefender deletes the scriptExpected false positive. Disable real-time protection, run MAS, re-enable.
MASHWID fails (0xC004C003)Try KMS38. Or MAS → [6] Troubleshoot → Fix Licensing.
MASOffice not detectedInstalled consumer (C2R) build, not Volume License. Rebuild config with PerpetualVL2021.
MASSays "180 days" not permanentGot Online KMS. Re-run, pick HWID for Windows, Ohook for Office.
MASActivation lost after hardware changeHWID tied to motherboard + CPU. Re-run HWID — re-binds automatically.
